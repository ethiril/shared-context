Catch-up on feature **$ARGUMENTS**. Protocol: `framework/README.md` §2 → Catch up.

**Format-protocol skip rule:** Open `.claude/settings.local.json` and check `shared_context_framework_version`. If it equals **2** (the current version, declared at README §7's `FRAMEWORK_VERSION`), the format note below is a complete substitute and **you can skip re-reading §7 entirely.** If missing or lower, read §7 in full, then update the field to 2 in settings.

**Format note (≈ README §7, v2):** Logs are per-event `.dsl` files in `log/` (one DSL line each). Repo statuses are `<iso>.positional`. Contracts are `<iso>-<repo>-v<X.Y.Z>.dsl`. Legacy `*.md`+YAML still parses — read both shapes.

**Read order — lead with the index, then drill down only as needed:**

1. **`features/$ARGUMENTS/_index.md` first.** Auto-generated by the renderer. Contains: mission status + latest checkpoint pointer + **excerpts from the latest snapshot's Headline + Open for the human + Next up + Where each repo stands** + recent decisions (last 5) + last 4 log entries (with summaries) + last 1 digest + latest positional per repo. ~85% of catch-up needs satisfied here in ~2K bytes.
2. `features/$ARGUMENTS/MISSION.md` — feature identity. (Skip if already loaded this session.)
3. Latest file in `features/$ARGUMENTS/orchestrator/` — **conditional**. Skip if `_index`'s embedded excerpts answered everything you need. Open the full snapshot only if you need sections `_index` doesn't embed (e.g. `## What shipped since the last snapshot`, `## Decisions made`) or if a referenced filename in the excerpts warrants the broader context.
4. `features/$ARGUMENTS/cursors/<your-repo>/current.md` (rolling). Fall back to latest timestamped file if `current.md` is missing (legacy).
5. `features/$ARGUMENTS/repos/<your-repo>/` latest positional — your task brief.
6. **Drill down only as needed**, using _index as the guide:
   - Open log entries newer than `cursor.last_log_read` that `_index` flagged as interesting (asks `to:` you, pivots, contract-changes affecting your code). Skip ones already summarised in _index.
   - For each API surface you'll actually touch: latest `contracts/<api>/*.dsl` (or legacy `.md`).
   - Decisions newer than `cursor.last_decision_read` with `status: accepted` — open only the ones the index summary suggests are relevant. Don't read every ADR by default.

**Skip** `status: superseded`, tombstoned files, and DSL entries retired by a later `[pv]`'s `supersedes:` (use `/audit $ARGUMENTS` for full history).

**Watch for in-session pivot triggers.** Before reporting state, scan the current conversation: if it reveals that the feature's direction has materially shifted (scope changed, a signed contract no longer holds, a core assumption broke), propose `/pivot $ARGUMENTS <reason>` before doing anything else.

Budget ≤ 5k tokens. Report state + next action.
